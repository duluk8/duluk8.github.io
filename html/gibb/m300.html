SSH-Setup

Use following commands for password less root login (on vmlp1): 

ssh-keygen -t ed25519 # (default 2x enter)
ssh-copy-id 192.168.220.11 # (vmls3 dns)
ssh 192.168.220.11
cat .ssh/authorized_keys # ssh key von vmadmin@vmLP1 kopieren
sudo vi /root/.ssh/authorized_keys # ssh key rein kopieren
ssh root@192.168.220.11

DNS-Setup

Installation & configuration

Install bind9 with following command (local without docker):
sudo apt install bind9
Now configure the bind files like shown below. Pay attention to the syntax if it isn't working try to debug with sudo named-checkconf /etc/bind/named.conf. It will show you the syntax error...
cd /etc/bind
vi named.conf.options

options {
    directory "/var/cache/bind";

    // If there is a firewall between you and nameservers you want
    // to talk to, you may need to fix the firewall to allow multiple
    // ports to talk.  See http://www.kb.cert.org/vuls/id/800113

    // If your ISP provided one or more IP addresses for stable 
    // nameservers, you probably want to use them as forwarders.  
    // Uncomment the following block, and insert the addresses replacing 
    // the all-0's placeholder.

    forwarders {
        8.8.8.8;
    };
    allow-recursion {
        192.168.210.0/24;
        192.168.220.0/24;
    };

    //========================================================================
    // If BIND logs error messages about the root key being expired,
    // you will need to update your keys.  See https://www.isc.org/bind-keys
    //========================================================================
    dnssec-validation auto;

    listen-on-v6 { any; };
};

Activate the changes

Restart the bind9 service with either bind9 or named and then test it with the following commands:
systemctl restart bind9
nslookup www.winter.camp 192.168.220.11
nslookup 192.168.220.1 192.168.220.11
If you need to debug the service this command cloud help: journalctl -xeu named.service

Configure clients

To make the dns work for your environment use following commands on servers (without GUI):
cd /etc/netplan
vi 00-eth0.yaml Important: search & addresses

network:          
version: 2        
renderer: networkd        
ethernets:        
    eth0:      
    addresses: [ 192.168.220.11/24 ]    
    gateway4: 192.168.220.1    
    nameservers:    
        search: [ winter.camp ]  
        addresses: [ 192.168.220.11 ]

On machines with GUI do this steps:
Open the Terminal:
sudo apt install network-manager-gnome
nm-connection-editor
Now select your ethernet (typically eth0) then go to IPv4 Settings and under Searchdomain type in: winter.camp then save. Now you need to disable and then enable your connection. So the changes take effect.

Integrate an additional hard drive

Look which disks are availlable:
sudo fdisk -l
Now look at something like /dev/sdb or /dev/sdc (choose the bigger one). Next, we partition the disc:
sudo fdisk /dev/sdc Then n (new partition), next p (primary), default, default, default, next w (write).
Next, we gonna format the /dev/sdc1 with:
sudo mkfs.ext4 /dev/sdc1
Use the /www directory as the mount directory (read carefully which directory).
mkdir /www
Next, we bind the partition: sudo mount /dev/sdc1 /www
To mount automatically by starting edit following file:
sudo vi /etc/fstab
Add following line: /dev/sdc1 /www ext4 defaults 0 2
For the changes to take effect run this command: mount -a
As a last step test it with df you should see something like this: /dev/sdc1 8153564 36 7717768 1% /www

Portainer

First create the docker volume: docker volume create portainer_data
Now start the container with docker run: docker run -d -p 8000:8000 -p 9443:9443 --name portainer --restart=always -v /var/run/docker.sock:/var/run/docker.sock -v portainer_data:/data portainer/portainer-ce:latest
Now you should see the container with docker ps and portainer should be available under https://192.168.220.11:9443. Now just login in and portainer is set up

Apache2

Place the web content (html files etc.) in the required directory (/www in this example). If it's a .tar.gz file extract it with: tar xvzf winter-camp.tar.gz
If there should be a owner for the directories (e.g. /www/www.winter.camp) then use following commands (all in the /www dir):
useradd -m operatingweb
groupadd wwwadmin
usermod -aG wwwadmin operatingweb
chown -R operatingweb:wwwadmin www.winter.camp ku1.winter.camp ku2.winter.camp
Next we navigate to the apache2 folder: cd /etc/apache2
Make gloub modifications with vi apache2.conf and add following code snippet (remove "):

<"Directory /www>
    Options Indexes FollowSymLinks
    AllowOverride None
    Require all granted
<"/Directory>

Next create the config files for each domain cd sites-available
cp 000-default.conf winter.camp.conf
Now modify the winter.camp.conf with vi winter.camp.conf and add:

<"VirtualHost *:80>
    ServerName www.winter.camp
    serveralias winter.camp

    ServerAdmin webmaster@localhost
    DocumentRoot /www/www.winter.camp

    ErrorDocument 404 /err/404.html

    ErrorLog ${APACHE_LOG_DIR}/www-error.log
    CustomLog ${APACHE_LOG_DIR}/www-access.log combined

<"/VirtualHost>

# vim: syntax=apache ts=4 sw=4 sts=4 sr noet

For additional sites do the same with for example ku1.winter.camp.conf and so on...

After that enable your sites with a2ensite winter.camp
a2ensite ku1.winter.camp
...
systemctl restart apache2
So now your changes should be active.
Test it with systemctl status apache2 and of course over the browser.
