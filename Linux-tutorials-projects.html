UFW konfigurieren (Ubuntu) Firewall

UFW konfigurieren (Ubuntu) Firewall

Schritt 1 — Verwenden von IPv6 mit UFW (optional)

Wenn auf Ihrem Ubuntu-Server IPv6 aktiviert ist, muss UFW so konfiguriert sein, dass IPv6 unterstützt wird, um Firewall-Regeln nicht nur für IPv4, sondern auch für IPv6 zu verwalten. Öffnen Sie dazu die UFW-Konfiguration mit nano oder Ihrem bevorzugten Editor.

sudo nano /etc/default/ufw

Stellen Sie dann sicher, dass der Wert von IPV6=yeslautet.

Schritt 2 — Einrichten von Standardrichtlinien

Auf Standardeinstellungen setzten:

sudo ufw default deny incoming
 
sudo ufw default allow outgoing

Schritt 3 — Zulassen von SSH-Verbindungen

Unbedingt vor Aktivierung einstellen, ansonsten ist kein SSH Zugriff möglich:

sudo ufw allow ssh

ufw wiess, was allow ssh bedeutet (Prot 22) ansonsten geht auch der command:

sudo ufw allow 22

oder welcher Port auch immer, je nachdem welchen SIe für SSH konfiguriert haben.

Schritt 4 — Aktivieren von UFW

sudo ufw enable

Y und Enter, um weiter zu fahren

#show status of ufw and the rules 
sudo ufw status verbose

starte danach das System neu, überprüfe mit dem obigen Befehl, dass Port 22 (SSH) allowed ist, wenn du in einer SSH Session bist.

22/tcp        ALLOW IN        Anywhere 

#reboot system
 
sudo reboot

Schritt 5 — Zulassen anderer Verbindungen

Beispiele:

#Spezifische IP-Adressen 
sudo ufw allow from 203.0.113.4 
sudo ufw allow from 203.0.113.4 to any port 22 

#Subnetze 
sudo ufw allow from 203.0.113.0/24 
sudo ufw allow from 203.0.113.0/24 to any port 22

 #sudo ufw allow from 203.0.113.0/24 to any port 22 ip addr

#Wenn Server eine öffentliche Netzwerkschnittstelle namens eth0 hat, könnten Sie HTTP-Verkehr (Port 80) dorthin zulassen 
sudo ufw allow in on eth0 to any port 80 
sudo ufw allow in on eth1 to any port 3306      #MySQL-Datenbankserver

Schritt 6 — Ablehnen von Verbindungen

#z.B. HTTP-Verbindungen ablehnen 
sudo ufw deny http 

#alle Verbindungen von <ip> ablehnen 
sudo ufw deny from 203.0.113.4

Schritt 7 — Löschen von Regeln

Wenn Regelnummern verwendet wurden:

#Liste von Firewall Regeln mit Nummer
sudo ufw status numbered

#delete nach nummer z.B. Regel 2
sudo ufw delete 2

Nach tatsächlicher Regel:

#Regel namens "allow http" 
sudo ufw delete allow http
 
#oder anstelle von http <allow 80> 
sudo ufw delete allow 80

Schritt 8 — Prüfen von UFW-Status und -Regeln

sudo ufw status verbose

Schritt 9 — Aktivieren oder Zurücksetzen von UFW

#disable ufw alle konfigurierten Einstellungen sind nicht länger aktivieret 
sudo ufw disable 

#reset alle regeln und man kann von neu starten 
sudo ufw reset
